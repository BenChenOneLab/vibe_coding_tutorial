Claude Code
AI 寫程式，你出想法

Anthropic 官方的 CLI 工具。在終端機裡跟 AI 對話，它幫你寫 code、改 bug、讀文件、跑指令。
不會寫程式也能用——用自然語言描述你要什麼就好。

三步安裝

1

裝 Node.js
到 nodejs.org 下載 LTS 版本安裝（需要 18+）。已經有的跳過。

2

安裝 Claude Code

npm install -g @anthropic-ai/claude-code
3

開始用
打開終端機，cd 到你的專案資料夾，輸入 claude，登入後就能開始對話。

第一次可以試什麼？

幫我看一下這個專案的架構 — 讓 AI 幫你讀 code

這段報錯了，幫我修 + 貼上錯誤訊息 — debug

@spec.md 根據這份規格幫我做一個表單 — 引用檔案給 context

/plan 我想加一個用戶通知系統 — 先規劃再執行

跟 ChatGPT 最大的差別：Claude Code 直接活在你的專案裡，能讀檔、改檔、跑指令。不用複製貼上程式碼，它直接幫你改。

01CLAUDE.md — 你的 AI 記憶

Claude Code 每次對話都是全新的。CLAUDE.md 就是讓它「記住你」的方式。
在專案根目錄建一個 CLAUDE.md，寫下你的偏好和規則，AI 每次啟動都會讀。

三層結構，由廣到細

🌎
Global — ~/.claude/CLAUDE.md
你的個人偏好，所有專案都吃得到
📁
Project — 專案根目錄/CLAUDE.md
專案架構、技術棧、團隊約定
📄
子目錄 — 任何子資料夾/CLAUDE.md
特定模組的細部規則（可選）

最小可用範例

# 語言 回覆用繁體中文，code 和 commit message 用英文 # 風格 Be terse. 不要囉唆 不要加我沒要求的功能 # 規則 改 code 前先讀完整個檔案 fix 失敗 2 次就換方向，不要微調

用了一陣子之後

.claude/rules/ — 拆出來的硬規則，可限定「碰到哪些檔案才生效」，不浪費 context

.claude/memory/ — Claude 自己累積的知識筆記，按需讀取不佔空間

/insights — 跑一次健檢報告，它會自動建議你該加哪些規則，複製貼上就好

不用一次寫完。先貼上面的範例，用幾天後覺得 AI 哪裡不順，就加一條規則。踩一次坑補一條，越用越順。

02Prompt 實戰技巧

跟 Claude Code 溝通，給 context 比給指令重要。

四個必學操作

1

@file 引用檔案
直接 @spec.md 把規格丟進去，不用複製貼上。可以一次引用多個檔案。

2

/plan 先規劃再執行
PM 最該用的功能。讓 AI 先列計畫給你看，你同意了才動手。避免它自作主張改錯方向。

3

迭代心法：三方案 → 挑一個 → 逐步加
先要求三個方案討論，挑最簡單的做出來，能用了再加功能。

4

錯誤處理：整段貼，不用看懂
跑出錯誤訊息？全選、複製、貼給它，說「這段報錯了，幫我修」。反覆修超過 3 次就說「換個做法重寫」。

範例 Prompt

好的 prompt 長這樣
我是體育博弈產品的 PM。
@betting-spec-template.md 這是我們目前的規格模板
@settlement-rules.csv 這是現有的結算規則表

幫我做一個 checklist 工具：選球種後自動帶出該球種的必填項目，逐項勾選確認，缺漏標紅。
先給我三個方案討論，不要直接寫 code。
心法：想像你在跟一個超強但剛入職的外包說話——它什麼都會做，但不知道你們公司的 context。你給越多背景，它做越準。

03真實案例：Frog AI

一個用 Claude Code 從零開發的 AI 穿搭內容生產系統。

CLAUDE.md 是踩坑踩出來的

80 行規則不是一天寫的，每條背後都是一個真實的坑。

第 1 週：基本規則（語言偏好、使用習慣）
第 2 週：AI 亂猜 bug 原因 → 加「先讀 code 再診斷，不准用猜的」
第 3 週：AI 說「完成了」但根本沒測 → 加「寫完必須用工具驗證，禁止腦內推演」
第 4 週：怕 WebFetch 撈到惡意指令 → 加 Prompt Injection 偵測提醒

踩坑實例：Recharts tooltip 位置跑掉，AI 用 CSS micro-tweak 修了 5 輪都沒用。最後我叫它換策略用內建 content prop，一次就好。從此加規則：fix 失敗 2 次就必須換方向。
Skills + Memory：讓 AI 有專長和記憶

Skills 給專業能力，Memory 讓 AI 跨 session 記住專案脈絡和已知坑。

Skills — 需要什麼裝什麼
frontend-design（UI 設計感）、ui-ux-pro-max（視覺打磨）、ltx-prompt / wan-prompt（影片 prompt 優化）— 共 4 個，每個都對應實際工作流。
Memory — 跨 session 的大腦
MEMORY.md 索引 + 7 個子檔案（已知坑、架構、執行流程、被糾正的記錄⋯）按需讀取，不浪費 context。
還有 Checkpoint Protocol：完成功能 → commit → review → retro → backup，一氣呵成。
Review & Retro ≠ Git Commit Log

Commit log 記「改了什麼」，Retro 記「為什麼這樣做」和「學到什麼」。

Git log：fix(tick-queue): stop infinite retry loop when ComfyUI is offline

RETRO.md：「autoTagger 原本用 SDK 需要 API key，.env 沒設導致 53 篇全 untagged。改用 claude -p CLI 零配置。Trade-off 是每次 spawn process 比較重，加 200ms delay 就夠。」

一個月 50+ 版本 RETRO 紀錄，按主題分類歸檔，commit 進 Git 就是團隊知識庫。
核心心法：把跟 AI 協作當成帶新人——CLAUDE.md 是 onboarding 文件，Memory 是工作筆記，Retro 是經驗傳承。踩一次坑就補一條，越用越順。

04Skills：可重複使用的能力包

把常做的工作流變成一鍵指令，像是幫 AI 裝外掛。

內建 Skills （點擊查看說明）

/commit
/commit
分析 staged changes，自動產生符合 conventional commits 格式的 commit message。
•
/review-pr
/review-pr
Review 當前分支的 PR，分析 diff、留 review comments，找 bug 和改進點。
•
/plan
/plan
進入 Plan Mode，Claude 只做調研和規劃，列出方案等你核准後才動手改 code。PM 必用。
•
/insights
/insights
分析過去 30 天的 session 紀錄，產出互動式 HTML 健檢報告，找出使用模式和改進建議。
•
/batch
/batch <instruction>
大規模重構利器。拆分任務成 5~30 個獨立單元，每個跑在獨立 worktree，平行執行後各自開 PR。
•
/simplify
/simplify [focus]
平行跑 3 個 review agent 檢查你改過的檔案，找出可重用、品質和效能問題後自動修復。

自訂 Skill — 三步驟變成 / 指令

1

建目錄
mkdir -p .claude/skills/write-prd

2

寫 SKILL.md（YAML frontmatter + 指令內容）

--- name: write-prd description: 用團隊模板撰寫 PRD allowed-tools: - Read - Write - WebFetch --- 依照 @prd-template.md 格式， 產出完整 PRD，包含目標、用戶故事、 驗收標準。$ARGUMENTS 是需求描述。

name = slash command 名稱，description = Claude 自動判斷何時載入的依據，$ARGUMENTS = 使用者傳入的參數

3

使用
打 /write-prd 新增用戶積分系統 即可觸發，或 Claude 判斷相關時自動載入。

進階控制：disable-model-invocation: true = 只有手動 /name 才觸發（適合 deploy 等高風險操作）。context: fork = 在獨立 subagent 執行，不污染主對話。

Skills 生態系

Product Manager Skills — 46 個現成的 PM skill，涵蓋 PRD 撰寫、用戶訪談分析、優先級排序、Sprint Planning 等。

SkillsMP — Agent Skills 市集，支援 Claude Code / Codex / Cursor 等多平台。alirezarezvani/claude-skills 有 192+ 個。

Skill 遵循 Agent Skills 開放標準，同一個 SKILL.md 可跨 Claude Code、Codex、Gemini CLI、Cursor 使用。

核心概念：CLAUDE.md 告訴 AI「你是誰」，Skills 告訴 AI「怎麼做」。兩者搭配就是你的個人 AI 工作系統。

05/insights 健檢報告

分析你過去 30 天的使用模式，產出互動式 HTML 報告。
像一個知識豐富的主管幫你做 code review——但 review 的是你跟 AI 協作的方式。

六階段分析流程

1
收集 — 讀取本機 session 日誌，過濾短對話
↓
2
摘要 — 長對話分塊，提取「要求什麼 / 做了什麼 / 卡在哪」
↓
3
切面提取 — 辨識深層目標、滿意度、摩擦點
↓
4
聚合 — 跨 session 找出模式：互動風格、出色工作流、反覆卡點
↓
5
執行摘要 — 什麼有效 / 什麼阻礙進度 / 快速勝利 / 進階建議
↓
6
HTML 報告 — 統計數據 + 視覺化圖表 + 可直接用的 CLAUDE.md 建議

報告會告訴你什麼？

任務完成率 — 完全達成 / 部分達成 / 未達成的比例

摩擦點分類 — 誤解需求、產出有 bug、過度修改⋯

互動風格 — 你是給詳細指令還是簡短提示後再修？

自動建議 CLAUDE.md 規則 — 根據你重複的指令自動生成

專案分佈 & 工具使用 — 你在哪些專案花最多時間

怎麼跑？報告在哪？

1

在 Claude Code 輸入 /insights，等幾分鐘（會分析 30 天內的 session）

2

跑完後自動產出互動式 HTML 報告並用瀏覽器開啟

3

報告存在本機：~/.claude/usage-data/report.html
完全本地運算，不上傳任何資料。隨時可回去打開這個檔案回顧。

建議每月跑一次。快取機制讓重複執行只分析新 session，追蹤自己跟 AI 協作的成長軌跡。

2026 新功能速覽

功能說明
Voice Mode
/voice		 按住空白鍵說話，放開送出。支援 20 種語言，不是「永遠在聽」的侵入式設計。用嘴巴跟 AI 協作。
/loop
/loop 5m ...		 定時重複執行指令。像輕量 cron job：/loop 5m check the deploy 每 5 分鐘幫你看一次部署狀態。
1M Context Opus 4.6 支援 100 萬 token 上下文窗口，可以一次丟進整個專案的程式碼。從 200K 直接跳到 1M。
MCP Servers 串接外部服務：Jira、Linear、Notion、Slack、PostHog、Amplitude⋯⋯不需要自己寫 API 串接。
Hooks 工具執行前後自動觸發行為。支援 HTTP hooks，可串接外部服務。例如：每次 commit 前自動跑 lint。
Computer Use AI 直接操作你的桌面：移動滑鼠、點按鈕、開瀏覽器。搭配 Remote Control 可以用手機遙控電腦上的 AI。
Scheduled Tasks 設定排程任務，定時自動執行。像是每天早上 9 點自動整理昨天的 PR 摘要。
